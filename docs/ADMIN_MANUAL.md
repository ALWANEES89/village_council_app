# ADMIN_MANUAL — دليل المشرف

> دليل استخدام لوحة الإدارة. الصلاحيات تُفرض في Firestore Rules — ما تراه يعتمد على دورك في المجلس. راجع `ROLES_AND_PERMISSIONS.md`.

## 1. الدخول للوحة الإدارة
- تظهر لوحة الإدارة لمن يملك صلاحية إدارية (`canOpenAdmin`).
- الأزرار والأقسام تظهر حسب صلاحياتك في **المجلس الحالي** فقط.

## 2. الأدوار (ملخّص)
| الدور | أبرز الصلاحيات |
|---|---|
| رئيس المجلس (chairman) | كل شيء (`fullAccess`) |
| مدير إداري (adminManager) | الأعضاء، الطلبات، الإعدادات، الحجوزات |
| المدير/المراجع المالي | مراجعة واعتماد/رفض الإيصالات |
| أمين السر (secretary) | مراجعة الطلبات، الإعلانات، الإشعارات، قراءة السجل |
| سوبر أدمن (منصّة) | إدارة كل المجالس |

## 3. طلبات الانضمام
- **مراجعة واعتماد/رفض** طلبات العضوية (`membershipRequests.review`/`members.approve`).
- الاعتماد يُنشئ عضوية بدور `member` تلقائياً + رقم عضو.
- **يُسجَّل في سجل الأحداث** (`membership_request.approved` + `membership.created`).

## 4. إدارة الأعضاء
- بحث/تصفية الأعضاء، وفتح تفاصيل العضو.
- **الإجراءات:** تفعيل، تعليق، إزالة، نقل بين المجالس.
- **تغيير الصلاحية (الدور):** يظهر الزر **فقط** لمن يملك `roles.manage`/`fullAccess` (رئيس المجلس/سوبر أدمن). يُسجَّل كـ `membership.role_changed`.
> إدارة الحالة (تفعيل/تعليق/إزالة) متاحة لمن يملك `members.manage`، لكنها **لا** تغيّر الدور/الصلاحيات.

## 5. مراجعة الإيصالات (المالية)
- قائمة الإيصالات قيد الاعتماد.
- **اعتماد** → تُعتمد الدفعة (`receipt.approved`). **رفض** (بسبب) → `receipt.rejected`.
- كلاهما يُسجَّل في سجل الأحداث بالفاعل ودوره.

## 6. مراجعة الحجوزات
- طلبات حجز المجلس (`canReviewBookings`: chairman/adminManager أو `bookings.approve`).
- **اعتماد/رفض** → `booking.approved`/`booking.rejected` في السجل.

## 7. إدارة الصلاحيات (الأدوار)
- تعديل صلاحيات كل دور في المجلس (`roles.manage`).
- التغيير يُسجَّل كـ `role.permissions_changed` مع القيم القديمة/الجديدة والفاعل.

## 8. سجل الأحداث (Audit Viewer)
- عرض **للقراءة فقط** لكل العمليات الحسّاسة في مجلسك.
- **فلاتر:** نوع العملية، الفاعل/الدور، نوع الهدف، نطاق التاريخ، بحث بالاسم.
- توسيع أي سجل يُظهر **القيمة السابقة/الجديدة** والمصدر.
- **لا يمكن تعديل أو حذف** أي سجل.
- مدير المجلس يرى سجل **مجلسه فقط**؛ السوبر أدمن يرى كل المجالس.

## 9. إدارة المجالس (سوبر أدمن فقط)
- **إضافة مجلس:** إنشاء مجلس جديد كامل البنية (أدوار/إعدادات/ملف مالي) دون التأثير على غيره.
- **تعديل/أرشفة/تفعيل** مجلس → يُسجَّل (`organization.updated`/`organization.status_changed`).
- تعيين مسؤول أولي (chairman) للمجلس الجديد.

## 10. مبادئ أمنية للمشرف
- لا تملك صلاحية خارج مجلسك (عزل تام).
- إخفاء الأزرار ليس أماناً — القواعد ترفض أي عملية غير مصرّح بها.
- كل عملية حسّاسة تُترك أثراً في سجل الأحداث الخادمي (لا يمكن تخطّيه أو حذفه).
