# قائمة QA الهاتف للنظام المالي

## إعداد آمن

- شغّل Firebase Emulator محليًا، ولا تستخدم مشروع `alrahmat-console`.
- Android Emulator: مرّر `FIREBASE_EMULATOR_HOST=10.0.2.2`.
- هاتف حقيقي: مرّر عنوان LAN لجهاز التطوير، ولا تضعه في الكود.
- تحقق من ظهور شريط «بيئة اختبار محلية» في debug ومن غيابه في release.

## السيناريوهات

- [ ] تبديل مجلسين للمستخدم نفسه وعدم ظهور بيانات المجلس السابق.
- [ ] مجلس مجاني بلا رسوم وهمية.
- [ ] اشتراك شهري واشتراك سنوي.
- [ ] الدفع عن النفس، وعن عضو واحد بالاسم، وعن عدة أعضاء.
- [ ] تطابق مبلغ الإيصال، ورفض الفرق والدفع الزائد.
- [ ] إدخال `٥` و`١٢٫٥٠٠` و`12.500`.
- [ ] رفع صورة وPDF ضمن الحد، ورفض النوع أو الحجم غير المسموح.
- [ ] انقطاع الشبكة بعد رفع الملف، ثم إعادة المحاولة دون معاملة أو ملف مكرر.
- [ ] اعتماد الإيصال ورفضه وسبب الرفض والتسديد الجزئي.
- [ ] حجز عضو ورسم العضو.
- [ ] حجز مستخدم غير عضو ورسم غير العضو ورفع إيصال لرسمه فقط.
- [ ] إلغاء pending، وطلب إلغاء approved، وقبول/رفض الإدارة.
- [ ] ظهور `refundRequired` دون استرداد تلقائي عند وجود مبلغ مدفوع.
- [ ] RTL والشاشات الصغيرة ولوحة المفاتيح وحالات loading/error/retry.
- [ ] اختبار Push على جهاز أو Emulator يدعم Google Play Services. لا يُستخدم Huawei P40 كتحقق FCM نهائي.

## أوامر مقترحة فقط

```powershell
flutter run --dart-define=USE_FIREBASE_EMULATORS=true --dart-define=FIREBASE_EMULATOR_HOST=10.0.2.2
flutter run --dart-define=USE_FIREBASE_EMULATORS=true --dart-define=FIREBASE_EMULATOR_HOST=<LAN_IP>
```

لا تُنفذ هذه الأوامر على هاتف متصل إلا بعد موافقة صريحة.
