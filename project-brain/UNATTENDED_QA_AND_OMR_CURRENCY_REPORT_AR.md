## 2. إصلاح فتح الإيصال

### السبب الجذري

كانت `getFinancialReceiptDownloadUrl` تستخدم `file.getSignedUrl()` داخل Functions Emulator. إنشاء Signed URL يحتاج بيانات service account مثل `client_email` ومفتاح توقيع، وهي غير متاحة في المحاكي المحلي، فظهر `firebase_functions/internal` رغم صحة الرفع والتفويض.

### الحل

- بقيت فحوص Auth و`organizationId` والعضوية وصلاحية المراجع وملكية المعاملة وتطابق `transactionId` مع مسار Storage قبل قراءة الملف.
- لا يُفعل مسار bytes إلا عندما تكون `FUNCTIONS_EMULATOR=true` خادميًا ويطابق project ID صراحة `demo-financial-prestaging`. لا يُقبل flag أو project ID من العميل.
- يستمر الإنتاج باستخدام Signed URL قصيرة العمر دون تغيير.
- يعيد المحاكي محتوى الملف بعد التفويض فقط، ويخزنه Flutter مؤقتًا في cache ثم يفتحه محليًا ويحاول حذفه عند مغادرة الشاشة.
- الحد الأقصى 10 MiB، مع التحقق من الاسم والامتداد و`contentType` وتوقيع PDF/JPEG/PNG/WebP. لا تُسجل bytes أو Base64.
- لم تتغير Firestore Rules أو Storage Rules، ولم تُضف download tokens، ولم تصبح الإيصالات عامة.

### اختبارات الأمان

- صاحب الإيصال يستطيع فتحه.
- المراجع المخول يستطيع فتحه عندما يختلف `membershipId` عن `userId`.
- المستخدم غير المخول ومجلس آخر يُرفضان.
- اختلاف `transactionId` أو Storage path يُرفض.
- الملف المفقود والحجم الزائد والنوع أو التوقيع غير المقبول تُرفض.
- مسار Emulator لا يعمل خارج `FUNCTIONS_EMULATOR`، ومسار الإنتاج ما زال Signed URL.
- لا يظهر محتوى الملف الخام في السجلات.

## 5. توحيد العملة

### السبب الجذري

كان تنسيق المبالغ موزعًا بين `formatBaisa` ونصوص يدوية و`toStringAsFixed(3)`، وكانت بعض الإشعارات تحفظ نصًا خامًا يحتوي عدد البيسات. أدى ذلك إلى اختلاف العرض وإمكان ظهور أعداد مثل 5000 أو كلمة «بيسة» للمستخدم.

### الحل المركزي

- `lib/core/formatters/omr_currency.dart`: تنسيق ثلاثي الخانات، fallback نصي للإشعار النظامي، parser للريال يعتمد الحساب الصحيح فقط.
- `lib/presentation/widgets/omr_amount.dart`: `OmrSymbol` و`OmrAmount` ومكونات الأسطر وحقول الإدخال الموحدة.
- `functions/omr_currency.js`: تنسيق الخادم وإنشاء payload مالي منظم.
- تقبل المدخلات الأرقام الإنجليزية والعربية بنوعيها، والفاصل `.` أو `,` أو `٫`، وفاصل الآلاف العربي `٬`. تُرفض أكثر من ثلاث خانات كسرية والقيم السالبة بدل تقريبها بصمت.
- أمثلة مثبتة: 5000 ← 5.000، و8000 ← 8.000، و12500 ← 12.500، و7500 ← 7.500، مع الرمز الرسمي في Flutter.
- الإشعارات الجديدة تحفظ `amountBaisa` و`currencyCode=OMR` و`bodyTemplate`، وتنسقها عند العرض. الإشعارات القديمة التي تحتوي عددًا خامًا وكلمة الوحدة تُحوّل وقت العرض للتوافق دون كسر البيانات القديمة.
- إشعار Android النظامي يستخدم مؤقتًا `ر.ع.` لأن نص النظام لا يستطيع تضمين أصل PNG؛ لا يستخدم كلمة «بيسة». داخل Flutter يُستخدم الرمز الرسمي المحلي.

### مواضع العرض المعدلة

- لوحة العضو الرئيسية وملخص الحساب والرسوم والاشتراكات.
- رفع الإيصال وسجل الإيصالات والخط الزمني للمعاملة.
- شاشة المراجع المالي والإدارة المالية ولوحة الإدارة.
- الدفع عن النفس وعن عضو واحد وعدة أعضاء والتوزيع المختلط.
- رسوم الحجز وإيصال الضيف.
- شاشة إشعارات التطبيق ورسائل الحالة والنجاح والخطأ المرتبطة بالمبالغ.
- التنسيق الخادمي للإشعارات المالية وseed جهاز QA.
